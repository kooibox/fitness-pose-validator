# 有效计数算法改进方案

## 背景

有效计数（Valid Count）用于统计动作标准的深蹲次数。原有的算法仅基于错误帧比例进行二元判断，存在以下局限性：

1. **无权重差异**：深度不足和膝盖内扣被同等对待，但后者风险更高
2. **忽略警告级别**：WARNING 级别的问题完全不影响有效性
3. **无连续错误惩罚**：偶尔出错和持续出错被同等对待
4. **缺乏反馈**：用户无法了解每次深蹲的具体得分

## 改进方案：加权错误分数系统

### 核心判断逻辑

**直观的关系：**

```
错误比例 < 有效阈值 → 有效 ✓
错误比例 >= 有效阈值 → 无效 ✗
```

**有效阈值的计算：**

```
有效阈值 = 基础容忍度 × (1 - 危险修正)

危险修正 = max(0, (平均错误权重 - 1.0) × 0.3)
平均错误权重 = 加权错误分数 / 错误帧数
```

### 有效区间示例（标准模式，基础容忍度=30%）

| 错误类型 | 平均权重 | 危险修正 | 有效阈值 | 判断 |
|---------|---------|---------|---------|------|
| 全是深度不足 | 1.0 | 0 | 30% | 错误比例<30%有效 |
| 全是背部弯曲 | 1.2 | 0.06 | 28.2% | 错误比例<28.2%有效 |
| 全是膝盖内扣 | 1.5 | 0.15 | 25.5% | 错误比例<25.5%有效 |
| 混合错误 | 1.3 | 0.09 | 27.3% | 错误比例<27.3%有效 |

### 错误类型权重

| 错误类型 | 权重 | 说明 |
|---------|------|------|
| 膝盖内扣 (KNEE_VALGUS) | 1.5 | 危险度高，可能导致受伤 |
| 背部弯曲 (BACK_BENT) | 1.2 | 中等风险，影响发力 |
| 深度不足 (DEPTH_INSUFFICIENT) | 1.0 | 基础权重，影响训练效果 |
| 速度过快 (TOO_FAST) | 0.5 | 低风险，主要影响控制 |

WARNING 级别按 0.3 的系数折算。

### 连续错误惩罚

当错误持续出现超过 3 帧时，每多一帧额外增加 0.1 的惩罚，调整后的错误比例：

```
调整后错误比例 = 错误比例 + 连续惩罚 × 0.1
```

### 容忍度阈值（按严格程度）

| 严格度 | 基础容忍度 | 说明 |
|--------|-----------|------|
| 宽松 | 50% | 错误比例超过 50% 才无效 |
| 标准 | 30% | 错误比例超过 30% 才无效 |
| 严格 | 15% | 错误比例超过 15% 就无效 |

## 代码实现

### 数据类

```python
@dataclass
class RepScore:
    """单次深蹲评分详情"""
    total_frames: int           # 下蹲状态总帧数
    error_frames: int           # 错误帧数
    error_ratio: float          # 错误帧比例
    weighted_error_score: float # 加权错误分数
    penalty_score: float        # 连续错误惩罚分数
    final_score: float          # 最终错误比例（含惩罚）
    quality_score: float        # 动作质量得分 (0-100)
    valid_threshold: float      # 有效阈值
    is_valid: bool              # 是否有效
```

### 核心计算逻辑

```python
def _calculate_rep_score(self) -> RepScore:
    # 1. 计算错误帧比例
    error_ratio = error_frames / total_frames
    
    # 2. 计算平均错误权重
    avg_error_weight = weighted_error_score / error_frames
    
    # 3. 计算危险修正和有效阈值
    danger_modifier = max(0, (avg_error_weight - 1.0) * 0.3)
    valid_threshold = base_tolerance * (1 - danger_modifier)
    
    # 4. 连续错误惩罚
    penalty_ratio = penalty_score / total_frames
    adjusted_error_ratio = error_ratio + penalty_ratio * 0.1
    
    # 5. 质量得分
    quality_score = max(0, (1 - adjusted_error_ratio) * 100)
    
    # 6. 判断有效性
    is_valid = adjusted_error_ratio < valid_threshold
```

## 示例场景

### 场景 1：同样错误比例，不同错误类型

假设下蹲 20 帧，5 帧错误（错误比例 25%），标准模式（基础容忍度 30%）：

| 错误类型 | 平均权重 | 有效阈值 | 调整后比例 | 有效 |
|---------|---------|---------|-----------|------|
| 5帧深度不足 | 1.0 | 30% | 25% | ✓ (25%<30%) |
| 5帧背部弯曲 | 1.2 | 28.2% | 25% | ✓ (25%<28.2%) |
| 5帧膝盖内扣 | 1.5 | 25.5% | 25% | ✓ (25%<25.5%) |

### 场景 2：连续错误 vs 分散错误

假设下蹲 20 帧，5 帧膝盖内扣错误：

| 场景 | 连续惩罚 | 调整后比例 | 有效阈值 | 有效 |
|------|---------|-----------|---------|------|
| 分散（1+1+1+1+1） | 0 | 25% | 25.5% | ✓ |
| 连续5帧 | 0.2 | 27% | 25.5% | ✗ |

### 场景 3：混合错误

假设下蹲 20 帧，3 帧深度不足 + 2 帧膝盖内扣：

```
加权分数 = 3×1.0 + 2×1.5 = 6.0
平均权重 = 6.0 / 5 = 1.2
危险修正 = (1.2 - 1.0) × 0.3 = 0.06
有效阈值 = 30% × (1 - 0.06) = 28.2%
错误比例 = 5/20 = 25%

25% < 28.2% → 有效 ✓
```
